# dns_debug.ps1

## Description
Captures and live-displays incoming DNS queries from Microsoft DNS Server debug logging.

The script enables DNS Server diagnostic logging to a timestamped log file, watches that file in real time, parses DNS packet header lines, decodes encoded query names such as `(3)www(7)example(3)com`, and shows a rolling console view of the most recent received queries.

Only received queries are displayed. Sent packets, localhost traffic, and unparsable log lines are ignored.

---

## Usage
Run the script in an elevated PowerShell session on a Windows DNS server:

```powershell
pwsh .\dns_debug.ps1
```

With custom log folder and a larger on-screen buffer:

```powershell
pwsh .\dns_debug.ps1 -LogFolder 'D:\dns_debug' -TailLines 50
```

Windows PowerShell also works if the DNS Server module is available:

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\dns_debug.ps1 -LogFolder 'C:\dns_debug' -TailLines 30
```

---

## Requirements
- Windows Server with the DNS Server role
- PowerShell with access to the `DnsServer` module
- Administrative privileges to run `Set-DnsServerDiagnostics`

External modules:
- None beyond the built-in DNS Server management cmdlets

Vendor-specific dependencies:
- Microsoft DNS Server

---

## Input / Output
- **Input:**
  - Optional `-LogFolder` parameter for the destination folder of the DNS debug log file
  - Optional `-TailLines` parameter for the number of recent received queries shown in the console
  - Live DNS debug log entries generated by Microsoft DNS Server

- **Output:**
  - A timestamped log file named like `dns-debug_yyyyMMdd-HHmmss.log`
  - A continuously refreshed console view with lines in this format:
    - `dd.MM.yyyy HH:mm:ss  <client-ip>  <qtype>  <qname>`

---

## Notes
- The script turns on extensive DNS debug logging, including UDP, TCP, full packets, receive/send packet logging, and query transaction logging.
- It is intended for short-term troubleshooting, not permanent operation.
- The script does **not** restore the previous DNS diagnostic settings when it exits. After use, you should manually reset logging to your normal server baseline.
- The parser only displays lines that match the expected DNS debug log pattern. Changes in log format may cause entries to be skipped.
- Only `Rcv` traffic is shown in the console output.
- Queries from `127.0.0.1` are filtered out.
- `TailLines` controls only the on-screen rolling buffer, not the size of the log file.
- `EnableLogFileRollover` is set to `$false` and `MaxMBFileSize` to `500`, so large troubleshooting sessions can create sizeable log files.
